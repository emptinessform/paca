# AID — 인간·AI 협업 시대의 소프트웨어 개발 방법론

**AID(Augmented Iterative Delivery)**는 AI 에이전트를 "도구"가 아니라 "팀원"으로 받아들이는 시대에 맞춰, 고전 소프트웨어 개발 방법론을 흡수해 정립한 **내재화 프레임워크**입니다.

오픈소스 프로젝트 관리 도구 **Paca**와 AWS의 **AI-DLC**를 분석하고, 고전 방법론(RUP·OOAD·CBD·SPL)의 유전자를 이식하여 **CMMI·한국 SP인증**과의 정합까지 설계한 종합 기술 백서를 담고 있습니다.

> 핵심 한 문장 — *"고전의 규율을 AI로 저비용 복원하고, 명세를 살아있게 하며, 책임은 사람에게 고정하라 — 그러면 협업·품질·인증이 한 방향으로 정렬된다."*

- **3. AID 정립** — 7원칙 · 4계층(리듬·규율·산출물·거버넌스) · 운영 모델 · AID ↔ AI-DLC 매핑

## 🔑 핵심 개념

- **P-A-C-A** — Plan · Act · Check · Adapt 반복 리듬
- **4계층** — 리듬(속도) · 규율(방향) · 산출물(진실의 원천) · 거버넌스(신뢰)
- **Living Spec** — 사람·AI가 공동 편집하는 단일 진실 원천(BDD + SDD)
- **Human Gate** — AI는 실행·자문(R·C), 최종 승인·책임(A)은 인간에 고정
- **인증 정합** — 협업의 부산물로 증적이 자동 생성 → CMMI·SP인증에 수렴
